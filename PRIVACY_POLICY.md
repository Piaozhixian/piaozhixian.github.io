# プライバシーポリシー

## 必須の理由

App Store Connectでアプリを提出する際、以下の理由でプライバシーポリシーが必要です：

1. **App Store審査ガイドライン**
   - すべてのアプリはプライバシーポリシーへのリンクを含める必要がある
   - App Store Connectのメタデータフィールドで必須項目

2. **権限使用の説明**
   - カメラ（NSCameraUsageDescription）を使用
   - フォトライブラリ（NSPhotoLibraryUsageDescription）を使用
   - これらの権限の使用目的を明確に説明する必要がある

3. **ユーザーの信頼**
   - プライバシーポリシーがあることで、ユーザーが安心してアプリを使用できる

## プライバシーポリシーの内容

以下の内容を含める必要があります：

### 1. 収集するデータの種類
- **写真・動画**: ユーザーが選択または撮影した画像・動画
- **アプリ内データ**: 運動計画、ステップ情報など（ローカル保存のみ）

### 2. データの使用方法
- 写真・動画: 運動ステップの表示・管理のみ（アプリ内でのみ使用）
- アプリ内データ: ローカルデバイスに保存、外部送信なし

### 3. データの保存場所
- すべてのデータはローカルデバイス（iPhone/iPad）にのみ保存
- クラウドサービスや外部サーバーへの送信なし

### 4. データの共有
- 第三者とのデータ共有なし
- 広告配信や分析ツールの使用なし

## プライバシーポリシーのテンプレート

```markdown
# プライバシーポリシー

最終更新日: [日付]

## はじめに

「カルトレ」（以下「本アプリ」）は、ユーザーのプライバシーを尊重し、個人情報の保護に努めます。

## 収集する情報

### 1. 写真・動画
本アプリは、運動ステップに画像や動画を添付するために、以下の権限を使用します：
- **カメラ**: 運動ステップの画像・動画を撮影するため
- **フォトライブラリ**: 既存の画像・動画を選択するため

これらの画像・動画は、アプリ内でのみ使用され、外部に送信されることはありません。

### 2. アプリ内データ
本アプリは、以下のデータをローカルデバイスに保存します：
- 運動計画の名称
- 運動ステップの情報（時間、名前、画像・動画）
- アプリの設定情報

これらのデータは、ユーザーのデバイスにのみ保存され、外部に送信されることはありません。

## データの使用目的

収集した情報は、以下の目的でのみ使用されます：
- 運動計画の作成・管理
- 運動ステップの表示・編集
- アプリ機能の提供

## データの保存場所

すべてのデータは、ユーザーのデバイス（iPhone/iPad）にのみ保存されます。
クラウドサービスや外部サーバーへの送信は一切行いません。

## データの共有

本アプリは、収集した情報を第三者と共有することはありません。
広告配信や分析ツールの使用も行いません。

## データの削除

ユーザーは、アプリを削除することで、すべてのデータを削除できます。
アプリ内のデータは、ユーザーが手動で削除することも可能です。

## お問い合わせ

プライバシーに関するご質問は、以下までお問い合わせください：
[連絡先メールアドレス]

## プライバシーポリシーの変更

本プライバシーポリシーは、予告なく変更される場合があります。
重要な変更がある場合は、アプリ内で通知します。
```

## 公開方法

### 方法1: GitHub Pages（無料・簡単）

1. GitHubリポジトリを作成
2. `privacy-policy.html` または `privacy-policy.md` を作成
3. GitHub Pagesを有効化
4. URL: `https://[ユーザー名].github.io/[リポジトリ名]/privacy-policy.html`

### 方法2: 個人サイト

既存のウェブサイトがある場合、そこに公開

### 方法3: 無料ホスティングサービス

- Netlify
- Vercel
- Firebase Hosting

## App Store Connectでの設定

1. App Store Connectにログイン
2. アプリを選択
3. 「App Privacy」セクションで以下を設定：
   - プライバシーポリシーURLを入力
   - 収集するデータの種類を選択：
     - 写真・動画: ユーザーコンテンツ
     - アプリ内データ: その他のデータ（ローカル保存のみ）

## 注意事項

- プライバシーポリシーは、アプリ提出前に公開されている必要がある
- URLは常にアクセス可能である必要がある
- 内容は実際のアプリの動作と一致している必要がある
